headless cms

a prototype cms for two — creators and developers.

01 · meta

2026 · spring boot · react · groovy · javascript

02 · what

built to close the gap between developers and content creators. a drag-and-drop visual editor for composing pages, live preview before publishing, and a headless api to feed any frontend — react, vue, mobile. extensible: drop in your own components with custom groovy hooks.

03 · why

most cmses force a tradeoff. headless ones are flexible but unfriendly to non-developers. visual builders trap you in their hosting. this prototype tries to keep both audiences happy — visual where it helps, api-first where it scales.

04 · how

spring boot backend exposing the headless api and persisting page trees. react frontend for the visual editor with live preview. groovy hook layer lets developers register custom components without rebuilding the editor. ships as a static demo on github pages so the editor experience is browsable end-to-end.
